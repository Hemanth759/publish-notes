## Overview

jOOQ (Java Object Oriented Querying) is a type-safe SQL builder that generates Java code from your database schema. It provides compile-time query validation and excellent performance.

## Key Features

- Type-safe SQL queries
- Code generation from database schema
- Full SQL support including advanced features
- Compile-time query validation
- Works alongside other ORMs
- Excellent performance
- Database-specific SQL dialect support
- Active record pattern support

## Entity Models

> [!important] How jOOQ Differs from JPA Frameworks
> Unlike [[Hibernate]] and [[Spring Data JPA]], jOOQ does **not** use JPA annotations. Instead, it:
> - **Generates code from your database schema** - Tables become Java classes
> - **Uses generated Record classes** - Type-safe representations of table rows
> - **Works with plain POJOs** - For mapping query results
>
> This means:
> - No `@Entity`, `@Table`, `@Column` annotations needed
> - No automatic relationship management
> - No automatic dirty checking (but Record classes track changes)
> - Database schema is the source of truth, not Java classes

### jOOQ's Two Approaches

jOOQ gives you two ways to work with data:

**1. Generated Record Classes** (Active Record pattern)
```java
// Generated by jOOQ from database schema
UsersRecord user = dsl.newRecord(USERS);
user.setUsername("john");
user.setEmail("john@example.com");
user.store();  // INSERT or UPDATE
```

**2. Plain POJOs** (for mapping results)
```java
// Your own POJO class
List<User> users = dsl.selectFrom(USERS)
    .fetchInto(User.class);  // Maps to POJO
```

### User Entity (Plain POJO)

```java
public class User {
    private Long id;
    private String username;
    private String email;
    private String status;
    private Long version;

    // Default constructor required for mapping
    public User() {}

    // Getters and setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    public String getStatus() { return status; }
    public void setStatus(String status) { this.status = status; }

    public Long getVersion() { return version; }
    public void setVersion(Long version) { this.version = version; }
}
```

### Order Entity (Plain POJO)

```java
public class Order {
    private Long id;

    private BigDecimal totalAmount;

    public LocalDateTime getOrderDate() { return orderDate; }
    public void setOrderDate(LocalDateTime orderDate) { this.orderDate = orderDate; }
```

### Generated Record Classes (by jOOQ)

jOOQ generates these from your database schema:

```java

public class UsersRecord extends UpdatableRecordImpl<UsersRecord> {
    public void setId(Long value) { ... }
    public Long getId() { ... }

    public void setUsername(String value) { ... }
    public String getUsername() { ... }

    public void setEmail(String value) { ... }
    public String getEmail() { ... }

    public void setStatus(String value) { ... }
    public String getStatus() { ... }

    public void setVersion(Long value) { ... }
    public Long getVersion() { ... }

    // Active Record methods
    public int store() { ... }  // INSERT or UPDATE
    public int delete() { ... }
    public void refresh() { ... }
}
```

### Key Differences from JPA

| Aspect | JPA (Hibernate/Spring Data JPA) | jOOQ |
|--------|--------------------------------|------|
| Source of Truth | Java classes with annotations | Database schema |
| Entity Definition | `@Entity` annotation required | Generated Record classes or plain POJOs |
| Column Mapping | `@Column(name = "...")` | Generated from schema, type-safe |
| Primary Key | `@Id` + `@GeneratedValue` | `.returning()` on INSERT |
| Relationships | `@OneToMany`, `@ManyToOne` | Manual JOIN queries |
| Foreign Keys | Object reference (`private User user`) | Simple field (`private Long userId`) |
| Type Safety | Runtime validation | Compile-time validation |
| Change Tracking | Automatic dirty checking | Record classes track changes |
| Query Building | JPQL strings or Criteria API | Type-safe DSL |

### Working with Relationships

jOOQ handles relationships through explicit JOINs:

```java
// Fetch users with their orders
Map<User, List<Order>> usersWithOrders = dsl
    .select()
    .from(USERS)
    .leftJoin(ORDERS).on(USERS.ID.eq(ORDERS.USER_ID))
    .fetchGroups(
        r -> r.into(USERS).into(User.class),
        r -> r.into(ORDERS).into(Order.class)
    );
```

### Converting Between Records and POJOs

```java
// Record to POJO
UsersRecord record = dsl.fetchOne(USERS, USERS.ID.eq(1L));
User user = record.into(User.class);

// POJO to Record
User user = new User();
user.setUsername("john");
UsersRecord record = dsl.newRecord(USERS, user);
record.store();
```

## Code Generation

jOOQ generates Java classes from your database schema:

```xml
<!-- Maven plugin configuration -->
<plugin>
    <groupId>org.jooq</groupId>
    <artifactId>jooq-codegen-maven</artifactId>
    <configuration>
        <jdbc>
            <url>jdbc:postgresql://localhost:5432/mydb</url>
            <user>user</user>
            <password>password</password>
        </jdbc>
        <generator>
            <database>
                <name>org.jooq.meta.postgres.PostgresDatabase</name>
                <includes>.*</includes>
            </database>
            <target>
                <packageName>com.example.generated</packageName>
                <directory>target/generated-sources/jooq</directory>
            </target>
        </generator>
    </configuration>
</plugin>
```

## Use Case Examples

### 1. Retrieve Records with Criteria

```java
public List<User> findActiveUsers(DSLContext dsl) {
    return dsl.selectFrom(USERS)
        .where(USERS.STATUS.eq("ACTIVE"))
        .fetchInto(User.class);
}

public List<User> findUsersByStatusAndEmail(DSLContext dsl, String status, String domain) {
    return dsl.selectFrom(USERS)
        .where(USERS.STATUS.eq(status)
            .and(USERS.EMAIL.like("%" + domain + "%")))
        .fetchInto(User.class);
}
```

### 2. Retrieve Records with JOIN

```java
public Map<User, List<Order>> findUsersWithOrders(DSLContext dsl) {
    return dsl.select()
        .from(USERS)
        .leftJoin(ORDERS).on(USERS.ID.eq(ORDERS.USER_ID))
        .where(USERS.STATUS.eq("ACTIVE"))
        .fetchGroups(
            r -> r.into(USERS).into(User.class),
            r -> r.into(ORDERS).into(Order.class)
        );
}
```

### 3. Insert New Data

```java
public User createUser(DSLContext dsl, String username, String email) {
    UsersRecord record = dsl.insertInto(USERS)
        .set(USERS.USERNAME, username)
        .set(USERS.EMAIL, email)
        .set(USERS.STATUS, "ACTIVE")
        .returning()
        .fetchOne();

    return record.into(User.class);
}
```

### 4. Update a Record

```java
public int updateUserEmail(DSLContext dsl, Long userId, String newEmail) {
    return dsl.update(USERS)
        .set(USERS.EMAIL, newEmail)
        .where(USERS.ID.eq(userId))
        .execute();
}
```

### 5. Transactional Write with Multiple Operations

```java
public void createUserWithOrders(DSLContext dsl, String username, String email,
                                List<OrderData> orderDataList) {
    dsl.transaction(configuration -> {
        DSLContext ctx = DSL.using(configuration);

        // Insert user
        UsersRecord userRecord = ctx.insertInto(USERS)
            .set(USERS.USERNAME, username)
            .set(USERS.EMAIL, email)
            .set(USERS.STATUS, "ACTIVE")
            .returning()
            .fetchOne();

        Long userId = userRecord.getId();

        // Insert orders
        for (OrderData orderData : orderDataList) {
            ctx.insertInto(ORDERS)
                .set(ORDERS.ORDER_NUMBER, orderData.getOrderNumber())
                .set(ORDERS.TOTAL_AMOUNT, orderData.getAmount())
                .set(ORDERS.ORDER_DATE, LocalDateTime.now())
                .set(ORDERS.USER_ID, userId)
                .execute();
        }

        // Transaction commits automatically if no exception
        // Rolls back if exception occurs
    });
}
```

### 6. Optimistic Locking

```java
// Using optimistic locking with version field
public void updateUserEmailOptimistic(DSLContext dsl, Long userId, String newEmail) {
    UsersRecord user = dsl.selectFrom(USERS)
        .where(USERS.ID.eq(userId))
        .fetchOne();

    if (user != null) {
        Long originalVersion = user.getVersion();

        int rowsUpdated = dsl.update(USERS)
            .set(USERS.EMAIL, newEmail)
            .set(USERS.VERSION, USERS.VERSION.add(1))
            .where(USERS.ID.eq(userId)
                .and(USERS.VERSION.eq(originalVersion)))
            .execute();

        if (rowsUpdated == 0) {
            throw new OptimisticLockException("User was modified by another transaction");
        }
    }
}
```

### 7. Pessimistic Locking

```java
// Pessimistic write lock
public void updateUserWithLock(DSLContext dsl, Long userId, String newEmail) {
    dsl.transaction(configuration -> {
        DSLContext ctx = DSL.using(configuration);

        // SELECT FOR UPDATE
        UsersRecord user = ctx.selectFrom(USERS)
            .where(USERS.ID.eq(userId))
            .forUpdate()
            .fetchOne();

        if (user != null) {
            user.setEmail(newEmail);
            user.update();
        }
        // Lock released on commit
    });
}

// Pessimistic read lock (shared lock)
public User getUserWithSharedLock(DSLContext dsl, Long userId) {
    return dsl.selectFrom(USERS)
        .where(USERS.ID.eq(userId))
        .forShare()
        .fetchOneInto(User.class);
}

// Lock with NOWAIT (fail immediately if locked)
public User getUserWithNoWait(DSLContext dsl, Long userId) {
    return dsl.selectFrom(USERS)
        .where(USERS.ID.eq(userId))
        .forUpdate()
        .noWait()
        .fetchOneInto(User.class);
}

// Lock with SKIP LOCKED (skip locked rows)
public List<User> getAvailableUsers(DSLContext dsl) {
    return dsl.selectFrom(USERS)
        .where(USERS.STATUS.eq("PENDING"))
        .forUpdate()
        .skipLocked()
        .fetchInto(User.class);
}
```

## Bulk Operations

jOOQ provides excellent performance for bulk operations - often the best choice for 10,000+ records.

### Method 1: Batch Insert (Recommended)

```java
public void bulkInsertUsers(DSLContext dsl, List<User> users) {
    dsl.transaction(configuration -> {
        DSLContext ctx = DSL.using(configuration);

        BatchBindStep batch = ctx.batch(
            ctx.insertInto(USERS, USERS.USERNAME, USERS.EMAIL, USERS.STATUS)
               .values((String) null, null, null)
        );

        for (User user : users) {
            batch.bind(user.getUsername(), user.getEmail(), user.getStatus());
        }

        batch.execute();
    });
}
```

### Method 2: Multi-row INSERT (Even Faster)

```java
public void bulkInsertUsersMultiRow(DSLContext dsl, List<User> users) {
    dsl.transaction(configuration -> {
        DSLContext ctx = DSL.using(configuration);

        InsertValuesStep3<UsersRecord, String, String, String> step =
            ctx.insertInto(USERS, USERS.USERNAME, USERS.EMAIL, USERS.STATUS);

        for (User user : users) {
            step = step.values(user.getUsername(), user.getEmail(), user.getStatus());
        }

        step.execute();
    });
}
```

### Batch Update

```java
public void bulkUpdateUsers(DSLContext dsl, List<User> users) {
    dsl.transaction(configuration -> {
        DSLContext ctx = DSL.using(configuration);

        BatchBindStep batch = ctx.batch(
            ctx.update(USERS)
               .set(USERS.EMAIL, (String) null)
               .set(USERS.STATUS, (String) null)
               .where(USERS.ID.eq((Long) null))
        );

        for (User user : users) {
            batch.bind(user.getEmail(), user.getStatus(), user.getId());
        }

        batch.execute();
    });
}
```

### Mixed Operations

```java
public void bulkMixedOperations(DSLContext dsl, List<User> usersToInsert,
                                List<User> usersToUpdate) {
    dsl.transaction(configuration -> {
        DSLContext ctx = DSL.using(configuration);

        // Combine different operations in one batch
        List<Query> queries = new ArrayList<>();

        // Add inserts
        for (User user : usersToInsert) {
            queries.add(
                ctx.insertInto(USERS)
                   .set(USERS.USERNAME, user.getUsername())
                   .set(USERS.EMAIL, user.getEmail())
                   .set(USERS.STATUS, user.getStatus())
            );
        }

        // Add updates
        for (User user : usersToUpdate) {
            queries.add(
                ctx.update(USERS)
                   .set(USERS.EMAIL, user.getEmail())
                   .set(USERS.STATUS, user.getStatus())
                   .where(USERS.ID.eq(user.getId()))
            );
        }

        ctx.batch(queries).execute();
    });
}
```

### Chunking for Large Datasets

```java
public void bulkInsertWithChunking(DSLContext dsl, List<User> users) {
    int chunkSize = 1000;

    for (int i = 0; i < users.size(); i += chunkSize) {
        int end = Math.min(i + chunkSize, users.size());
        List<User> chunk = users.subList(i, end);

        bulkInsertUsers(dsl, chunk);
    }
}
```

## Advanced Features

### Complex Queries

```java
// Subqueries
public List<User> findUsersWithHighValueOrders(DSLContext dsl, BigDecimal threshold) {
    return dsl.selectFrom(USERS)
        .where(USERS.ID.in(
            dsl.select(ORDERS.USER_ID)
               .from(ORDERS)
               .where(ORDERS.TOTAL_AMOUNT.gt(threshold))
        ))
        .fetchInto(User.class);
}

// Window functions
public List<UserRank> getUserRanksByOrderTotal(DSLContext dsl) {
    return dsl.select(
            USERS.USERNAME,
            sum(ORDERS.TOTAL_AMOUNT).as("total"),
            rank().over().orderBy(sum(ORDERS.TOTAL_AMOUNT).desc()).as("rank")
        )
        .from(USERS)
        .join(ORDERS).on(USERS.ID.eq(ORDERS.USER_ID))
        .groupBy(USERS.ID, USERS.USERNAME)
        .fetchInto(UserRank.class);
}

// Common Table Expressions (CTE)
public List<User> findUsersWithCTE(DSLContext dsl) {
    CommonTableExpression<Record2<Long, BigDecimal>> cte = name("user_totals")
        .as(dsl.select(ORDERS.USER_ID, sum(ORDERS.TOTAL_AMOUNT).as("total"))
               .from(ORDERS)
               .groupBy(ORDERS.USER_ID));

    return dsl.with(cte)
        .select(USERS.fields())
        .from(USERS)
        .join(cte).on(USERS.ID.eq(cte.field(ORDERS.USER_ID)))
        .where(cte.field("total", BigDecimal.class).gt(BigDecimal.valueOf(1000)))
        .fetchInto(User.class);
}
```

```java
public void bulkInsertWithCopy(DSLContext dsl, List<User> users) {

// MySQL ON DUPLICATE KEY UPDATE
public void upsertUser(DSLContext dsl, User user) {
    dsl.insertInto(USERS)
        .set(USERS.USERNAME, user.getUsername())
        .set(USERS.EMAIL, user.getEmail())
        .set(USERS.STATUS, user.getStatus())
        .onDuplicateKeyUpdate()
        .set(USERS.EMAIL, user.getEmail())
        .set(USERS.STATUS, user.getStatus())
        .execute();
}
```

## When to Use jOOQ

- You need type-safe SQL
- Working with complex SQL queries
- Want compile-time query validation
- Database-first approach
- Performance is critical
- Need full SQL feature support
- Want to leverage database-specific features
- Building data-intensive applications

## Performance Considerations

- Excellent performance for bulk operations (~200-400ms for 10k records)
- Type-safe queries prevent runtime errors
- Minimal overhead
- Full control over SQL generation
- Can use database-specific optimizations
- Efficient result mapping
- Best choice for bulk operations alongside MyBatis

## Common Pitfalls

- Requires code generation step
- Learning curve for API
- Generated code can be large
- Schema changes require regeneration
- Less abstraction than full ORMs
- Need to understand SQL well
